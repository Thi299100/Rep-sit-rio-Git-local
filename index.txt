    <h1>GIT - VERSIONAMENTO</h1>
    <p>Atividade de aprendizado sobre controle de versão com Git</p>
    
    <h2>Comandos Git Principais:</h2>
    <ul>
        <li><strong>git status</strong> - Verifica o status do repositório e mostra arquivos modificados</li>
        <li><strong>git add</strong> - Adiciona arquivos para a área de staging (preparação para commit)</li>
        <li><strong>git commit -m "mensagem"</strong> - Cria um commit com as mudanças e uma mensagem descritiva</li>
        <li><strong>git push</strong> - Envia commits para o repositório remoto</li>
        <li><strong>git pull</strong> - Baixa atualizações do repositório remoto</li>
        <li><strong>git merge</strong> - Mescla branches (ramificações) do projeto</li>
        <li><strong>git checkout</strong> - Muda de branch ou restaura arquivos</li>
        <li><strong>git remote</strong> - Gerencia conexões com repositórios remotos</li>
    </ul>

    <h2>O que é Versionamento?</h2>
    <p>Versionamento é o processo de controlar e registrar todas as mudanças feitas em um projeto ao longo do tempo, permitindo que você volte a versões anteriores se necessário.</p>
